# Security Policy

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

If you discover a security concern in this project, report it privately through GitHub's private vulnerability reporting:

- Open the [private vulnerability reporting form](https://github.com/franklesniak/EFingPlanner/security/advisories/new).

> **Maintainers:** Private vulnerability reporting must be enabled in GitHub settings (**Settings → Code security**) for the link above to work. This repository publishes no security email address; GitHub private vulnerability reporting is the intake channel.

### What to Include

When reporting, please include:

- A clear description of the concern
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested fixes or mitigations (if known)

### Response Timeline

We will acknowledge receipt of your report within **48 hours** and aim to provide a more detailed response within **7 days**, including:

- Confirmation of the issue
- An assessment of severity
- An estimated timeline for a fix

### Disclosure Policy

We follow coordinated disclosure practices:

1. **Private Report:** Report the concern privately using the method above
2. **Assessment:** We assess the concern and determine severity
3. **Fix Development:** We develop and test a fix
4. **Release:** We release the fix and, where appropriate, publish a security advisory
5. **Public Disclosure:** After the fix is released, details may be disclosed publicly

We appreciate your help in keeping this project secure!

## Supported Versions

This is an evolving, Markdown-only educational-content repository. Only the current published content on the default branch is maintained; please refer to the latest version.
